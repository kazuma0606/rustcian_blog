# v4 Azure Resource Boundaries

This document describes which Azure services are used in v4, what each is
responsible for, and any known limitations or planned improvements.

## Resource map

```
Azure Subscription
└── Resource Group: rustacian-<env>-rg  (japaneast)
    ├── App Service Plan: rustacian-<env>-asp  (Linux, B1/P1v3)
    ├── Linux Web App: rustacian-<env>-app
    │     Identity: SystemAssigned managed identity
    │     Container: rustacian-blog-backend (Docker)
    │     Auth: Entra ID (admin routes only; public routes unauthenticated)
    ├── Storage Account: rustacian<env>st  (Standard LRS)
    │     Table: comments   (PartitionKey=post_slug, RowKey=comment_id)
    │     Table: contacts   (PartitionKey="contact",  RowKey=message_id)
    ├── Key Vault: rustacian-<env>-kv  (RBAC)
    │     Secret: appinsights-connection-string  (auto-populated by Terraform)
    │     Secret: slack-webhook-url              (set manually post-apply)
    │     Secret: azure-openai-api-key           (set manually post-apply)
    │     Secret: storage-account-key            (set manually post-apply)
    ├── Log Analytics Workspace: rustacian-<env>-law
    ├── Application Insights: rustacian-<env>-ai
    ├── Azure OpenAI: rustacian-<env>-oai
    │     Deployment: gpt-4o-mini (Standard, 10K TPM)
    └── Communication Service: rustacian-<env>-acs  (reserved, not active)
```

## Blog content delivery

Static blog pages are **not** served from the App Service.
The static site (`dist/`) is generated by the `generate-static` CLI subcommand
and deployed separately to Azure Static Web Apps or Blob Storage + CDN by the
content repository's GitHub Actions workflow.

The App Service serves:
- Admin UI (`/admin/*`) — requires Entra ID auth
- Comment and contact form endpoints (`/posts/*/comments`, `/contact`)
- Search API (`/search`) — server-side rendered via Tantivy
- Image proxy (`/images/*`)
- JSON API (`/posts`, `/posts/{slug}`)

## Known limitations

### Table Storage authentication

The `table.rs` client currently authenticates via **SharedKeyLite** using the
storage account key stored as a Key Vault secret (`storage-account-key`).
The Terraform IaC also provisions a `Storage Table Data Contributor` role
assignment for the App Service managed identity in anticipation of migrating
to **managed identity auth**, which would remove the need for the account key
entirely. This migration is deferred to a future phase.

### Azure OpenAI region

`gpt-4o-mini` availability varies by region. If `japaneast` does not have the
model available, change `location` in `variables.tf` to `eastus` or `westus`.
See the [Azure OpenAI model availability table](https://learn.microsoft.com/azure/ai-services/openai/concepts/models).

### Communication Services

The `comms` module provisions an Azure Communication Services resource in the
`Asia Pacific` data location. It is not wired into the application in v4.
Email notification support is planned for a future phase.
